Get A DOM element at a specific index in an array of elements.

:::info

The querying behavior of this command matches exactly how
[`.eq()`](https://api.jquery.com/eq) works in jQuery.

:::

## Syntax

```javascript
.eq(index)
.eq(indexFromEnd)
.eq(index, options)
.eq(indexFromEnd, options)
```

### Usage

**<Icon name="check-circle" color="green" /> Correct Usage**

```javascript
cy.get('tbody>tr').eq(0) // Yield first 'tr' in 'tbody'
cy.get('ul>li').eq(4) // Yield fifth 'li' in 'ul'
```

**<Icon name="exclamation-triangle" color="red" /> Incorrect Usage**

```javascript
cy.eq(0) // Errors, cannot be chained off 'cy'
cy.getCookies().eq(4) // Errors, 'getCookies' does not yield DOM element
```

### Arguments

**<Icon name="angle-right" /> index** **_(Number)_**

A number indicating the index to find the element at within an array of
elements. Starts with 0.

**<Icon name="angle-right" /> indexFromEnd** **_(Number)_**

A negative number indicating the index position from the end to find the element
at within an array of elements.

**<Icon name="angle-right" /> options** **_(Object)_**

Pass in an options object to change the default behavior of `.eq()`.

| Option    | Default                                                              | Description                                                                              |
| --------- | -------------------------------------------------------------------- | ---------------------------------------------------------------------------------------- |
| `log`     | `true`                                                               | Displays the command in the [Command log](/guides/core-concepts/cypress-app#Command-Log) |
| `timeout` | [`defaultCommandTimeout`](/guides/references/configuration#Timeouts) | Time to wait for `.eq()` to resolve before [timing out](#Timeouts)                       |

### Requirements [<Icon name="question-circle"/>](/guides/core-concepts/introduction-to-cypress#Chains-of-Commands)

- `.eq()` requires being chained off a command that yields DOM element(s).

### Assertions [<Icon name="question-circle"/>](/guides/core-concepts/introduction-to-cypress#Assertions)

- `.eq()` will automatically [retry](/guides/core-concepts/retry-ability) until
  it
  [finds an element](/guides/core-concepts/introduction-to-cypress#Default-Assertions).

### Timeouts [<Icon name="question-circle"/>](/guides/core-concepts/introduction-to-cypress#Timeouts)

- `.eq()` can
  [time out waiting](/guides/core-concepts/introduction-to-cypress#Default-Assertions)
  for the subject to have an element at `index`.

### Yields [<Icon name="question-circle"/>](/guides/core-concepts/introduction-to-cypress#Subject-Management)

- `.eq()` is a query command, and will not break a chain of
  [retry-ability](/guides/core-concepts/retry-ability#Commands-Queries-and-Assertions).
  It is _safe_ to chain further commands that use the subject after `.eq()`.
- `.eq()` yields the element from the subject at `index`.

## Examples

### Index

#### Find the 2nd element within the elements

```html
<ul>
  <li>tabby</li>
  <li>siamese</li>
  <li>persian</li>
  <li>sphynx</li>
  <li>burmese</li>
</ul>
```

```javascript
cy.get('li').eq(1).should('contain', 'siamese') // true
```

#### Make an assertion on the 3rd row of a table

```html
<table>
  <tr>
    <th>Breed</th>
    <th>Origin</th>
  </tr>
  <tr>
    <td>Siamese</td>
    <td>Thailand</td>
  </tr>
  <tr>
    <td>Sphynx</td>
    <td>Canada</td>
  </tr>
  <tr>
    <td>Persian</td>
    <td>Iran</td>
  </tr>
</table>
```

```javascript
cy.get('tr').eq(2).should('contain', 'Canada') //true
```

### Index From End

#### Find the 2nd from the last element within the elements

```html
<ul>
  <li>tabby</li>
  <li>siamese</li>
  <li>persian</li>
  <li>sphynx</li>
  <li>burmese</li>
</ul>
```

```javascript
cy.get('li').eq(-2).should('contain', 'sphynx') // true
```
