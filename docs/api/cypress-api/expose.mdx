# Cypress.expose

`get` and `set` exposed public configuration _in your tests_.

`Cypress.expose()` provides synchronous access to configuration that is safe to expose in the browser context. It may be used for feature flags, plugin configuration, and other public values that need to be available outside of Cypress command chains.

:::caution

**Security considerations**

`Cypress.expose()` is intended for **public/non-sensitive** configuration values only. Values set with `Cypress.expose()` are accessible in the browser context, including to application code, third-party scripts, and browser extensions.

**Do NOT** store sensitive data (API keys, passwords, tokens, etc.) using `Cypress.expose()`. Use [`cy.env()`](/api/commands/env) or [`cy.task()`](/api/commands/task) for sensitive values.

:::

## Syntax

```javascript
Cypress.expose()
Cypress.expose(key)
Cypress.expose(key, value)
Cypress.expose(object)
```

### Arguments

<Icon name="angle-right" /> **key _(String)_**

The name of the exposed configuration variable to get or set.

<Icon name="angle-right" /> **value _(any)_**

The value of the exposed configuration variable to set. Can be any type (string, number, boolean, object, etc.).

<Icon name="angle-right" /> **object _(Object)_**

Set multiple exposed configuration variables with an object literal. Values are merged with existing values.

## Examples

### No Arguments

#### Get all exposed configuration variables

:::cypress-config-example

```ts
{
  expose: {
    pluginConfig: 'value1',
    featureFlag: true,
    apiVersion: 2,
  },
}
```

:::

```javascript
Cypress.expose() // => {pluginConfig: 'value1', featureFlag: true, apiVersion: 2}
```

### Key

#### Return a single exposed configuration variable

:::cypress-config-example

```ts
{
  expose: {
    pluginConfig: 'my-plugin-config',
    featureFlag: true,
  },
}
```

:::

```javascript
Cypress.expose('pluginConfig') // => "my-plugin-config"
Cypress.expose('featureFlag') // => true
Cypress.expose('nonExistent') // => undefined
```

### Key and Value

#### Set a single exposed configuration variable

:::caution

**Scope**

Remember, any changes that you make to exposed configuration variables using this API will only be in effect for the remainder of the tests _in the same spec file._

:::

:::cypress-config-example

```ts
{
  expose: {
    pluginConfig: 'initial-value',
  },
}
```

:::

```javascript
Cypress.expose('pluginConfig', 'updated-value')

Cypress.expose('pluginConfig') // => "updated-value"
```

### Object

#### Set multiple exposed configuration variables at once

:::cypress-config-example

```ts
{
  expose: {
    pluginConfig: 'initial-value',
    featureFlag: false,
  },
}
```

:::

```javascript
Cypress.expose({
  pluginConfig: 'updated-value',
  featureFlag: true,
})

Cypress.expose() // => {pluginConfig: 'updated-value', featureFlag: true}
```

### Configuration File

#### Set exposed configuration variables in `cypress.config.js`

Exposed configuration variables can be set in your Cypress configuration file:

:::cypress-config-example

```javascript
{
  expose: {
    pluginConfig: 'foo',
    featureFlag: true,
    apiVersion: 1,
  }
}
```

:::

```javascript
// In your test or support file
const pluginConfig = Cypress.expose('pluginConfig') // => "foo"
const featureFlag = Cypress.expose('featureFlag') // => true
const apiVersion = Cypress.expose('API_VERSION') // => 1
```

### CLI Flags

#### Set exposed configuration variables via command line

You can set exposed configuration variables using the `--expose` or `-x` CLI flags:

```shell
cypress run --expose pluginConfig=foo,featureFlag=true,API_VERSION=1
```

Or using the short form:

```shell
cypress run -x pluginConfig=foo,featureFlag=true,API_VERSION=1
```

:::info

**CLI Override**

Values set via CLI flags override values set in the configuration file.

:::

### Plugin Use Case

#### Using `Cypress.expose()` in a plugin

`Cypress.expose()` is particularly useful for plugins that need synchronous access to configuration values outside of `cy` command contexts:

```javascript
// cypress/support/commands.js or plugin file
const pluginConfig = Cypress.expose('pluginConfig')

if (pluginConfig === 'enabled') {
  // Initialize plugin with configuration
  initializePlugin(pluginConfig)
}
```

#### Migrating from `Cypress.env()` to `Cypress.expose()`

If you're migrating a plugin from `Cypress.env()` to `Cypress.expose()`, the API is similar:

**Before (using deprecated `Cypress.env()`):**

```javascript
const config = Cypress.env('PLUGIN_CONFIG')
```

**After (using `Cypress.expose()`):**

```javascript
const config = Cypress.expose('PLUGIN_CONFIG')
```

Update your configuration file:

**Before:**

```ts
{
  env: {
    PLUGIN_CONFIG: 'value',
  },
}
```

**After:**

```ts
{
  expose: {
    PLUGIN_CONFIG: 'value',
  },
}
```

## Notes

### When to use `cy.env()` vs `Cypress.expose()`

<CypressEnvVsCypressExpose />

See [`cy.env()`](/api/commands/env) for more details on when to use it.

### Scope Limitations

- Exposed configuration variables set in the browser context are only available as long as the browser is open
- Changes made to exposed configuration variables persist for the remainder of tests in the same spec file
- Changes do NOT persist past the browser into the Node.js context (e.g., `setupNodeEvents`, `cy.task()` handlers)

### Security Considerations

Since `Cypress.expose()` values are accessible in the browser context:

- **Never** store sensitive credentials or secrets
- Values can be accessed by application code, third-party scripts, and browser extensions
- Values can be inspected in browser developer tools
- Use [`cy.env()`](/api/commands/env) for sensitive values that should remain privileged

### Configuration Priority

Exposed configuration variables are resolved in the following order (later sources override earlier ones):

1. Configuration file (`expose` key in `cypress.config.{js,ts,mjs,cjs}`)
2. CLI flags (`--expose` or `-x`)

Runtime changes made via `Cypress.expose(key, value)` or `Cypress.expose(object)` override both configuration file and CLI values for the remainder of the spec file.
