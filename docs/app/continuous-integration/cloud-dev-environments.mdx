# Running Cypress in Cloud AI Development Environments

:::info

##### <Icon name="question-circle" color="#4BBFD2" /> What you'll learn

- How to run Cypress in cloud AI development environments
- How to configure network access for Cypress downloads
- How screenshots and video behave in these environments

:::

Cloud AI development environments — such as
[Claude Code on the web](https://claude.ai/code) — let you write, run, and
debug code entirely through an AI assistant in a remote sandboxed container.
Cypress `run` mode works in these environments with one important prerequisite:
network access must be configured to allow Cypress to download its binary.

## Network configuration

Cypress downloads its binary during installation. In cloud AI environments,
outbound network traffic is typically routed through a sandbox proxy, which may
require you to explicitly allow new domains before they can be reached.

Make sure your environment allows outbound access to the required Cypress URLs
before running Cypress. For the full list, see
[Allow Cypress URLs on VPNs](/app/references/troubleshooting#Allow-Cypress-URLs-on-VPNs).

For Claude Code on the web specifically, new domains trigger a permission prompt
the first time they are accessed. See the
[Claude Code on the web — Network access](https://code.claude.com/docs/en/claude-code-on-the-web#network-access)
documentation for how to configure domain permissions in that environment.

## Screenshots

Screenshots work as expected. On failure, Cypress automatically saves a
screenshot to the
[`screenshotsFolder`](/app/references/configuration#Screenshots) (default:
`cypress/screenshots`) inside the remote workspace.

In AI coding environments, screenshots are particularly valuable because the AI
can read the captured images directly to diagnose test failures — no additional
tooling required.

## Video recording

Video recording is **disabled by default** and should stay that way. While
Cypress can capture video, the file is written to the remote filesystem and
cannot be retrieved once the session ends. Leave `video` at its default of
`false` and rely on screenshots and
[Test Replay](/cloud/features/test-replay) instead.
