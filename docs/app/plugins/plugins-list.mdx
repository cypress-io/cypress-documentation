# Cypress Plugins

import PluginsList from '@site/src/components/plugins-list'

Plugins extend and customize how you write, run, and report on your Cypress
tests. They add capabilities like bundling and preprocessors, custom commands,
network and API helpers, visual, accessibility, and AI-assisted testing, CI
integrations, and rich reporters. They're published as versioned npm modules, so
you can adopt and update them independently of Cypress itself.

This page curates plugins built by Cypress and the community. Each is labeled
with a trust badge and live signals (latest version, supported Cypress versions,
and how recently it was updated) so you can choose one with confidence. Built
something useful?
[Submit your own plugin](https://github.com/cypress-io/cypress-documentation/blob/master/CONTRIBUTING.md#adding-plugins).

:::info

<strong>Looking for the API docs?</strong>

Check out our [How to use Plugins](/app/plugins/plugins-guide) or our
[Node Events Overview](/api/node-events/overview) for writing a plugin.

:::

## Choosing a trustworthy plugin

Each plugin is labeled with a badge showing who maintains it and whether Cypress
has reviewed it. See the badge key below.

:::caution

Community plugins are contributed by third parties and are **not vetted,
audited, or endorsed by Cypress**. You install and run them at your own risk.
Before adding one to your project, take a moment to evaluate it.

:::
