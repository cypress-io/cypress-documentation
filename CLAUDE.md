# Claude Memory

## Code Comments

- Prefer NOT to add code comments when the code is self-explanatory.
- Always prefer no comment over a verbose or redundant one — none over verbosity.
- Only add a comment when it explains *why* something is done (non-obvious intent, a workaround, or a tricky edge case), not *what* the code is doing.
- Match the comment density and style of the surrounding code.
